## 3. 필요한 API 키 / 서비스 설정

### Firebase
- **Console**: https://console.firebase.google.com
- **설정 파일**: `firebase-config.js`
- **필요 서비스**: Authentication, Firestore, Storage
- **승인 도메인 추가 필요**: `landing01-iota.vercel.app`

### Google Gemini API (챗봇용)
- **발급처**: https://aistudio.google.com/app/apikey
- **사용자가 직접 입력**: 챗봇 설정 버튼 클릭 → API 키 입력
- **저장 위치**: localStorage (`plantcare_gemini_api_key`)

### 토스페이먼츠 (결제)
- **개발자센터**: https://developers.tosspayments.com
- **설정 파일**: `checkout.html`, `payment-success.html`
- **필요 키**:
  - Client Key (클라이언트용)
  - Secret Key (서버용 - 보안 주의!)

## 4. 로컬 개발 서버 실행

### 방법 1: VS Code Live Server
1. VS Code에서 프로젝트 폴더 열기
2. Live Server 확장 설치
3. `index.html` 우클릭 → "Open with Live Server"

### 방법 2: Python 간이 서버
```bash
# Python 3
python -m http.server 8000

# 브라우저에서 http://localhost:8000 접속
```

### 방법 3: Node.js serve
```bash
# serve 설치 (최초 1회)
npm install -g serve

# 서버 실행
serve .

# 브라우저에서 http://localhost:3000 접속
```

## 8. 보안 주의사항

### 절대 커밋하면 안 되는 것들
- Firebase API 키 (이미 `firebase-config.js`에 있음 - 공개용)
- 토스페이먼츠 Secret Key
- 기타 비밀 키

### 환경 변수로 관리해야 할 것들 (Vercel Dashboard)
- `TOSS_SECRET_KEY` - 토스페이먼츠 시크릿 키
- `GEMINI_API_KEY` - Gemini API 키 (선택적)

## 9. 문제 해결

### Google 로그인이 안 될 때
1. Firebase Console → Authentication → Settings → Authorized domains
2. `landing01-iota.vercel.app` 추가 확인

### 챗봇이 응답하지 않을 때
1. 챗봇 설정(⚙️) 클릭
2. Gemini API 키 입력 확인
3. 브라우저 콘솔에서 에러 확인

### 결제가 안 될 때
1. `checkout.html`에서 토스 Client Key 확인
2. 토스페이먼츠 개발자센터에서 키 상태 확인
